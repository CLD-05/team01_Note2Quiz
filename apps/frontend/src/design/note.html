Docker 아키텍처 및 핵심 개념 정리

도커(Docker)는 애플리케이션을 컨테이너라는 표준화된 유닛으로 패키징하여, 어디서든 일관되게 실행할 수 있도록 돕는 오픈소스 플랫폼입니다. 기존 가상 머신(VM)과 달리 호스트 OS의 커널을 공유하며 프로세스를 격리하기 때문에 가볍고 빠릅니다.

1. Docker Engine의 구조

도커 엔진은 클라이언트-서버 모델을 따릅니다. 주요 구성 요소로는 Docker Daemon (dockerd), REST API, 그리고 Docker CLI가 있습니다. 사용자가 CLI를 통해 명령어를 입력하면 REST API를 통해 데몬에 전달되고, 데몬은 이미지 빌드, 컨테이너 실행 등의 작업을 수행합니다.

2. 이미지 레이어링 시스템

도커 이미지는 읽기 전용(Read-only) 레이어의 적층 구조로 이루어져 있습니다. Dockerfile의 각 명령어는 새로운 레이어를 생성하며, 변경되지 않은 레이어는 캐시되어 재사용됩니다. 컨테이너가 실행될 때 최상단에 쓰기 가능한 컨테이너 레이어(Writable Layer)가 추가되어 변경 사항을 저장합니다.

3. 컨테이너 라이프사이클

컨테이너는 생성(Created), 실행(Running), 일시정지(Paused), 중지(Stopped), 삭제(Deleted)의 상태를 가집니다. 'docker run' 명령어는 사실상 create와 start 명령어를 합친 것이며, 프로세스가 종료되면 컨테이너도 자동으로 중지 상태로 전환됩니다.

이러한 아키텍처 덕분에 개발 환경과 운영 환경의 차이를 최소화할 수 있으며, 마이크로서비스 아키텍처(MSA)를 구현하는 데 필수적인 도구로 자리 잡았습니다.
